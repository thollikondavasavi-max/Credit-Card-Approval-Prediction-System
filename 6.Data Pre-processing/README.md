# Data Pre-processing

Data pre-processing is an essential step in the machine learning pipeline to ensure data quality, consistency, and model performance. The following pre-processing techniques were applied in this project:

## 1. Drop Duplicate Features

Duplicate records and redundant features were identified and removed from the dataset to eliminate bias and improve model efficiency. Removing duplicate data helps prevent overfitting and ensures that the model learns from unique observations.

## 2. Handling Missing Values

Missing values were detected and treated using appropriate techniques such as removing incomplete records or replacing missing entries with statistical measures such as mean, median, or mode depending on the feature type and data distribution.

## 3. Data Cleaning and Merging

The dataset was cleaned by correcting inconsistencies, removing irrelevant columns, standardizing formats, and merging related datasets when necessary. This process ensured a structured and reliable dataset for analysis and model training.

## 4. Feature Engineering

New features were created from existing attributes to improve the predictive capability of the machine learning models. Feature engineering helps capture hidden patterns and relationships within the data that may not be directly available in the original dataset.

## 5. Handling Categorical Values

Categorical variables were transformed into numerical representations using encoding techniques such as Label Encoding and One-Hot Encoding. This conversion allows machine learning algorithms to process categorical information effectively.

After completing these pre-processing steps, the dataset became clean, consistent, and suitable for training and evaluating machine learning models.
